# deriv-ux-writing

The source of truth for Deriv's UX writing standards — voice, tone, 
terminology, and reusable copy. Maintained by the UX team.

## Contents
- `SKILL.md` — the UX writing guideline, structured so it can also be 
  consumed programmatically (e.g. by tooling that generates or checks 
  copy against Deriv standards)
- `product-glossary.md` — approved product terminology
- `binance-crypto-glossary.md` — crypto and P2P terminology
- `copy-bricks.csv` — reusable copy components

## Ownership
This content is owned and maintained by the UX team. Changes should go 
through PR review so the writing standards stay consistent.
